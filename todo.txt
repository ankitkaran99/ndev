# ndev Project Summary

## Goal

Build **`ndev`**, a Python-based CLI tool for managing multiple compiled PHP-FPM versions on Debian systems.

Primary target:

* Debian Forky
* Debian Trixie
* Debian Sid
* Other Debian-based distributions (future)

Each PHP version is compiled from source and installed independently without affecting the system PHP.

---

# Objectives

* Install multiple PHP versions side by side.
* Compile PHP from source.
* Run multiple PHP-FPM instances simultaneously.
* Keep installations isolated from the system.
* Provide a developer-friendly CLI similar to `nvm`, `rustup`, or `sdkman`.

---

# Installation Layout

```
~/.ndev/

├── cache/
├── downloads/
├── builds/
├── chroot/
├── logs/
├── run/
├── php/
│   ├── 8.3/
│   ├── 8.4/
│   └── 8.5/
│
├── current -> php/8.4
│
└── config.toml
```

Each installed PHP version contains its own:

```
bin/
sbin/
lib/
etc/
include/
var/
```

No files are installed into:

```
/usr
/etc
/usr/local
```

---

# Build Environment

Use a persistent build environment matching the host Debian release.

Example:

```
Host: Debian Forky

↓

Build Environment:
~/.ndev/chroot
```

The build environment is reused across all PHP versions.

---

# Runtime

Each PHP version runs independently.

Example:

```
PHP 8.4
Socket:
/run/ndev/php84.sock

PHP 8.5
Socket:
/run/ndev/php85.sock
```

Multiple PHP-FPM versions can run simultaneously.

# CLI Commands

## Installation

```
ndev install 8.4
ndev install 8.5
ndev uninstall 8.4
```

---

## Runtime

```
ndev start 8.4
ndev stop 8.4
ndev restart 8.4
ndev reload 8.4
ndev status 8.4
```

---

## Version Management

```
ndev list

ndev available

ndev current

ndev use 8.4
```

---

## Maintenance

```
ndev doctor

ndev clean

ndev update

ndev shell
```

---

## Logs

```
ndev logs 8.4
```

---

## Extensions

```
ndev ext list 8.4

ndev ext install redis 8.4

ndev ext uninstall redis 8.4

ndev ext enable redis 8.4

ndev ext disable redis 8.4
```

# Build Workflow

1. Resolve requested version (e.g. `8.4` → latest `8.4.x`).
2. Download PHP source archive.
3. Reuse cached downloads when available.
4. Build inside the persistent build environment.
5. Install into `~/.ndev/php/<version>`.
6. Generate runtime configuration (`php.ini`, `php-fpm.conf`, `www.conf`).
7. Record build metadata (version, configure options, compiler, build date).
8. Start and manage PHP-FPM instances independently.

---

# Design Principles

* Python orchestrates the entire workflow.
* PHP is always compiled from source.
* Each PHP version is isolated.
* No modification of system PHP.
* Reusable build environment.
* Modular architecture with clear separation of responsibilities.
* Maintainable, testable, and extensible codebase.
* CLI experience inspired by tools like `nvm`, `rustup`, and `sdkman`.
